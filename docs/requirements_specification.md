# 要求仕様書
## 馬力計測モバイルアプリ「HorsepowerTracker」

| 項目 | 内容 |
|------|------|
| ドキュメント種別 | 要求仕様書 |
| アプリ名 | HorsepowerTracker |

---

## 1. 目的

本ドキュメントは、モバイルアプリ「HorsepowerTracker」の要求仕様を定義するものである。本アプリは、ユーザーが自車両の馬力をスマートフォン上で手軽に計測・記録・分析できる環境を提供することを目的とする。

---

## 2. 対象プラットフォーム

| 項目 | 内容 |
|------|------|
| リリース対象OS | Android、iOS |
| Android バージョン | Android 14 以上（API レベル 34 以上） |
| iOS バージョン | iOS 17.0 以上 |

---

## 3. アーキテクチャ要件

| 項目 | 内容 |
|------|------|
| アーキテクチャパターン | MVVM（Model-View-ViewModel） |
| 状態管理 | Provider |
| Widget 方針 | すべて StatelessWidget を使用 |
| ローカルデータベース | SQLite（端末内ローカル保存） |

---

## 4. 基本機能要件

本アプリが提供すべき根幹機能を以下に示す。

- 馬力の計測を行えること
- 計測した記録の表示を行えること
- 車両の登録ができること
- 課金機能を要すること



---

## 6. 非機能要件

- テキスト表示はローカライゼーション対応（日本語、英語）とする
- データは端末ローカルに保存し、外部サーバへの送信は行わない
- GPS（緯度・経度・標高）情報へのアクセスを行う
